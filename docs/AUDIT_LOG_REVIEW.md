# Huong dan xem va phan tich Audit Log

## 1. Audit log duoc luu o dau?

Audit log duoc luu trong bang `audit_logs` cua co so du lieu **PostgreSQL**, cung container voi du lieu chinh cua he thong.

**Pham vi ghi log:** He thong chi ghi cac hanh dong bien doi du lieu (POST, PUT, PATCH, DELETE) tren cac endpoint admin (`/api/v1/admin/...`). Cac request GET (doc du lieu) va cac endpoint khong phai admin khong duoc ghi vao bang nay.

---

## 2. Cau truc bang `audit_logs`

```sql
create table audit_logs (
    id          bigint generated by default as identity primary key,
    created_at  timestamp(6) with time zone not null,  -- thoi diem hanh dong xay ra
    actor       varchar(255),                          -- email/username cua admin thuc hien
    actor_role  varchar(255),                          -- role cua actor, phan cach bang dau phay
    action      varchar(255) not null,                 -- loai hanh dong (xem danh sach ben duoi)
    request_method varchar(16) not null,               -- HTTP method: POST, PUT, PATCH, DELETE
    request_path   varchar(512) not null,              -- duong dan API duoc goi
    entity_type varchar(255),                          -- loai doi tuong bi tac dong (vi du: products, orders)
    entity_id   varchar(255),                          -- ID cua doi tuong bi tac dong
    ip_address  varchar(128),                          -- IP cua admin (xem luu y o muc 5)
    payload     jsonb                                  -- tham so request (da an cac truong nhay cam)
);
```

**Index da duoc tao:**

| Index | Cot | Muc dich |
|---|---|---|
| `idx_audit_logs_created_at` | `created_at DESC` | Query theo khoang thoi gian |
| `idx_audit_logs_actor` | `actor` | Tim hoat dong theo user |
| `idx_audit_logs_entity` | `(entity_type, entity_id)` | Tim lich su thay doi cua mot doi tuong |

**Cac gia tri `action` co the gap:**

| Gia tri | Khi nao duoc ghi |
|---|---|
| `create` | POST tao moi mot entity |
| `update` | PUT hoac PATCH cap nhat entity |
| `delete` | DELETE xoa entity |
| `import` | POST import du lieu hang loat |
| `changePassword` | Doi password qua admin |
| `createNotification` | Tao thong bao moi |

**Truong `payload`:** Chua cac tham so request duoi dang JSON. Cac truong nhay cam (`password`, `token`, `secret`, `apiKey`, `accessKey`, `checksum`) duoc tu dong thay bang `[REDACTED]` truoc khi luu.

## 3. Ket noi vao database de chay query

### Qua Docker Compose (cach thong dung nhat)

```bash
# Chay tu thu muc goc cua repo (noi co file .env)
source .env 2>/dev/null || true
docker compose exec postgres \
  psql -U "${POSTGRES_USER:-app}" -d "${POSTGRES_DB:-app_db}"
```

Hoac chay query mot dong ma khong can vao shell interactive:

```bash
docker compose exec postgres \
  psql -U "${POSTGRES_USER:-app}" -d "${POSTGRES_DB:-app_db}" \
  -c "<cau-lenh-SQL>"
```

### Qua psql tu host (neu PostgreSQL co port mo ra ngoai)

```bash
psql "postgresql://app:<password>@127.0.0.1:5432/app_db"
```

## 4. Cac truy van SQL mau

### 4.1. 50 hanh dong gan nhat cua admin

```sql
SELECT
    created_at AT TIME ZONE 'Asia/Ho_Chi_Minh' AS local_time,
    actor,
    actor_role,
    action,
    request_method,
    request_path,
    entity_type,
    entity_id,
    ip_address
FROM audit_logs
ORDER BY created_at DESC
LIMIT 50;
```

### 4.2. Toan bo hoat dong cua mot user cu the

Thay `admin@example.com` bang email can tim:

```sql
SELECT
    created_at AT TIME ZONE 'Asia/Ho_Chi_Minh' AS local_time,
    action,
    request_method,
    request_path,
    entity_type,
    entity_id,
    ip_address
FROM audit_logs
WHERE actor = 'admin@example.com'
ORDER BY created_at DESC
LIMIT 200;
```

### 4.3. Lich su thay doi cua mot entity cu the

Tim tat ca hanh dong lien quan den mot san pham, don hang... co ID cu the:

```sql
SELECT
    created_at AT TIME ZONE 'Asia/Ho_Chi_Minh' AS local_time,
    actor,
    action,
    request_method,
    ip_address,
    payload
FROM audit_logs
WHERE entity_type = 'products'   -- doi thanh: orders, users, warranties, ...
  AND entity_id   = '42'         -- doi thanh ID thuc te
ORDER BY created_at DESC;
```

### 4.4. Thong ke so hanh dong theo IP (phat hien bat thuong)

```sql
SELECT
    ip_address,
    COUNT(*) AS total_actions,
    COUNT(DISTINCT actor) AS distinct_actors,
    MIN(created_at) AS first_seen,
    MAX(created_at) AS last_seen
FROM audit_logs
WHERE created_at >= NOW() - INTERVAL '24 hours'
  AND ip_address IS NOT NULL
GROUP BY ip_address
ORDER BY total_actions DESC
LIMIT 30;
```

### 4.5. Hanh dong xoa du lieu (DELETE) trong 7 ngay qua

```sql
SELECT
    created_at AT TIME ZONE 'Asia/Ho_Chi_Minh' AS local_time,
    actor,
    actor_role,
    entity_type,
    entity_id,
    ip_address,
    payload
FROM audit_logs
WHERE action = 'delete'
  AND created_at >= NOW() - INTERVAL '7 days'
ORDER BY created_at DESC;
```

### 4.6. Hanh dong import hang loat

```sql
SELECT
    created_at AT TIME ZONE 'Asia/Ho_Chi_Minh' AS local_time,
    actor,
    request_path,
    ip_address,
    payload
FROM audit_logs
WHERE action = 'import'
ORDER BY created_at DESC
LIMIT 50;
```

### 4.7. Thong ke hanh dong theo ngay (30 ngay qua)

```sql
SELECT
    DATE(created_at AT TIME ZONE 'Asia/Ho_Chi_Minh') AS ngay,
    action,
    COUNT(*) AS so_lan
FROM audit_logs
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY ngay, action
ORDER BY ngay DESC, so_lan DESC;
```

### 4.8. Xuat CSV de gui cho bo phan ky thuat/bao mat

Chay tu terminal cua may chu (yeu cau psql co quyen ghi file):

```bash
docker compose exec postgres \
  psql -U "${POSTGRES_USER:-app}" -d "${POSTGRES_DB:-app_db}" \
  -c "\COPY (
    SELECT
      created_at AT TIME ZONE 'Asia/Ho_Chi_Minh' AS local_time,
      actor,
      actor_role,
      action,
      request_method,
      request_path,
      entity_type,
      entity_id,
      ip_address
    FROM audit_logs
    WHERE created_at >= NOW() - INTERVAL '30 days'
    ORDER BY created_at DESC
  ) TO STDOUT WITH CSV HEADER" > audit_export_$(date +%Y%m%d).csv
```

File CSV se duoc luu tai thu muc hien tai cua may chu voi ten dang `audit_export_20260423.csv`.

## 5. Luu y quan trong ve IP Address

Truong `ip_address` chi ghi dung IP thuc cua admin **khi bien moi truong duoc cau hinh dung**:

| Bien | Gia tri can thiet | Hau qua neu thieu |
|---|---|---|
| `APP_AUDIT_TRUST_FORWARDED_FOR` | `true` | `ip_address` luu toan bo la `127.0.0.1` (IP cua Nginx) thay vi IP thuc cua client |

Kiem tra gia tri hien tai:

```bash
grep APP_AUDIT_TRUST_FORWARDED_FOR .env
```

Neu dang la `false` hoac bi comment, xem huong dan doi sang `true` tai:
[docs/DEPLOYMENT_GUIDE.md — muc 6 Cau hinh bat buoc khi chay sau Reverse Proxy](DEPLOYMENT_GUIDE.md#6-cau-hinh-bat-buoc-khi-chay-sau-reverse-proxy-production-checklist)

> **Luu y:** Log da ghi voi IP sai khong the sua lai. Sau khi doi `APP_AUDIT_TRUST_FORWARDED_FOR=true` va restart backend, cac ban ghi moi se ghi dung IP. Cac ban ghi cu van giu gia tri `127.0.0.1`.

## 6. Don dep log cu (neu can)

Audit log tang dan theo thoi gian va khong co co che tu dong xoa. De don log cu khi bang qua lon:

```sql
-- Xem dung luong hien tai cua bang
SELECT
    pg_size_pretty(pg_total_relation_size('audit_logs')) AS total_size,
    COUNT(*) AS total_rows,
    MIN(created_at) AS oldest_entry,
    MAX(created_at) AS newest_entry
FROM audit_logs;
```

```sql
-- Xoa log cu hon 1 nam (nen backup truoc)
DELETE FROM audit_logs
WHERE created_at < NOW() - INTERVAL '365 days';
```

> Chay `VACUUM ANALYZE audit_logs;` sau khi xoa hang loat de Postgres thu hoi dung luong.
